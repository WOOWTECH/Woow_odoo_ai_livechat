## 簡介

**即時聊天 N8N 整合**模組將 Odoo 18 的即時聊天系統與 n8n(一個強大的工作流程自動化平台)連接。此整合讓您能夠:

- 當訪客發送訊息時自動觸發 n8n 工作流程
- 將 AI 驅動的自動回應發送回聊天會話
- 建立複雜的聊天自動化工作流程(常見問題機器人、工單建立、CRM 更新等)
- 追蹤所有 webhook 活動以進行監控和除錯

### 運作方式

1. **訪客發送訊息** → Odoo 在即時聊天中接收
2. **Odoo 觸發 webhook** → 將訊息資料發送到 n8n 工作流程
3. **n8n 處理訊息** → 執行您的自訂工作流程(AI、資料庫查詢等)
4. **n8n 發送回應** → 將自動回覆發送回 Odoo
5. **訪客看到回應** → 訊息出現在聊天視窗中

### 使用案例

- **AI 聊天機器人**: 連接到 ChatGPT、Claude 或其他 AI 服務以提供智慧回應
- **FAQ 自動化**: 在轉接給客服人員之前自動回答常見問題
- **潛在客戶資格審查**: 收集訪客資訊並自動建立 CRM 潛在客戶
- **工單建立**: 在外部系統中產生支援工單
- **多頻道通知**: 透過 Slack、Discord 或電子郵件通知您的團隊
- **業務邏輯**: 根據訪客訊息執行複雜的工作流程

## 安裝

### 前置需求

在安裝此模組之前,請確保您有:

1. 已安裝並執行 **Odoo 18.0 或更高版本**
2. 已安裝**即時聊天模組**(通常預設包含)
3. 可從您的 Odoo 伺服器存取的 **n8n 實例**(雲端或自行託管)

### 安裝步驟

#### 步驟 1: 安裝模組

1. 將模組下載或複製到您的 Odoo 附加元件目錄:
   ```bash
   cd /path/to/odoo/addons
   git clone https://github.com/WOOWTECH/im_livechat_n8n.git
   ```

2. 重新啟動您的 Odoo 伺服器以偵測新模組

3. 以管理員身分登入 Odoo

4. 前往**應用程式**選單

5. 點擊**更新應用程式列表**

6. 搜尋「Live Chat N8N」

7. 點擊**安裝**

#### 步驟 2: 驗證安裝

安裝後,您應該會看到:

- 即時聊天頻道設定中的新「N8N 整合」分頁
- 網站 → 設定下的新「Webhook 日誌」選單項目

## 設定

### 設定即時聊天頻道

#### 步驟 1: 開啟頻道設定

1. 前往**網站** → **即時聊天** → **頻道**
2. 選擇現有頻道或建立新頻道
3. 點擊 **N8N 整合**分頁

#### 步驟 2: 啟用整合

1. 勾選**啟用 N8N 整合**核取方塊
2. **API 金鑰**欄位將自動填入安全的隨機金鑰
3. 複製並儲存此 API 金鑰 - 您在設定 n8n 時會需要它

#### 步驟 3: 設定 Webhook URL

1. 在 **N8N Webhook URL** 欄位中輸入您的 n8n webhook URL

   範例 URL:
   - 雲端 n8n: `https://your-instance.app.n8n.cloud/webhook/odoo-livechat`
   - 自行託管: `https://n8n.yourdomain.com/webhook/odoo-livechat`
   - 本地測試: `http://localhost:5678/webhook/odoo-livechat`

2. 點擊**儲存**

**重要注意事項**:
- 在正式環境中使用 HTTPS 以確保安全
- 如果使用 HTTP(localhost 除外),模組會發出警告
- 確保 URL 可從您的 Odoo 伺服器存取

#### 步驟 4: 測試連線

1. 點擊**測試 Webhook 連線**按鈕
2. 如果一切設定正確,您應該會看到成功通知
3. 檢查您的 n8n 工作流程以驗證它是否收到測試事件

### API 金鑰管理

#### 檢視 API 金鑰

API 金鑰顯示在 **N8N 整合**分頁中。請妥善保管!

#### 重新產生 API 金鑰

如果您需要輪換 API 金鑰(建議定期執行):

1. 點擊**重新產生 API 金鑰**按鈕
2. 複製新金鑰
3. 使用新金鑰更新您的 n8n 工作流程
4. 舊金鑰將立即停止運作

**何時重新產生**:
- 定期安全輪換(建議每 90 天)
- 如果 API 金鑰遭到洩露或曝光
- 移除特定 n8n 工作流程的存取權限時

## 設定 n8n 工作流程

### 基本工作流程結構

Odoo 即時聊天的典型 n8n 工作流程包含三個元件:

1. **Webhook 觸發器** - 從 Odoo 接收訊息
2. **處理邏輯** - 您的自訂工作流程(AI、資料庫等)
3. **HTTP 請求** - 將回應發送回 Odoo

### 逐步工作流程建立

#### 步驟 1: 建立 Webhook 觸發器

1. 在 n8n 中建立新工作流程
2. 新增 **Webhook** 節點
3. 設定 webhook:
   - **HTTP 方法**: POST
   - **路徑**: 選擇唯一路徑(例如 `odoo-livechat`)
   - **驗證**: 無(我們將在回應中使用 API 金鑰)

4. 儲存並啟動工作流程
5. 複製 webhook URL(例如 `https://your-n8n.com/webhook/odoo-livechat`)
6. 將此 URL 貼到您的 Odoo 頻道設定中

#### 步驟 2: 新增處理邏輯

範例 1: **簡單自動回覆**

新增 **Set** 節點以建立靜態回應:

```javascript
// 節點: Set Response
{
  "response": "感謝您聯絡我們!客服人員將盡快為您服務。"
}
```

範例 2: **AI 驅動的回應**

新增 **OpenAI** 節點(或類似節點):

1. 新增 **OpenAI** 節點
2. 使用您的 API 金鑰進行設定
3. 使用傳入的訊息: `{{ $json.message.body }}`
4. 取得 AI 回應並傳遞到下一個節點

範例 3: **FAQ 資料庫查詢**

新增 **MySQL** 或 **HTTP 請求**節點以查詢您的 FAQ 資料庫:

```sql
SELECT answer FROM faq
WHERE LOWER(question) LIKE LOWER('%{{ $json.message.body }}%')
LIMIT 1
```

#### 步驟 3: 將回應發送到 Odoo

新增設定如下的 **HTTP 請求**節點:

**設定**:
- **方法**: POST
- **URL**: `{{ $json.metadata.callback_url }}` 或您的 Odoo URL + `/im_livechat_n8n/webhook`

**標頭**:
```json
{
  "Content-Type": "application/json",
  "X-API-Key": "your-api-key-from-odoo"
}
```

**主體**(JSON):
```json
{
  "action": "send_message",
  "session_uuid": "{{ $json.session.uuid }}",
  "message": {
    "body": "{{ $json.response }}",
    "author_name": "支援機器人"
  }
}
```

**提示**:
- 將 `your-api-key-from-odoo` 替換為來自 Odoo 頻道的實際 API 金鑰
- 使用表達式來引用前一個節點的資料
- 您可以自訂 `author_name`(例如「AI 助理」、「FAQ 機器人」)

### 完整工作流程範例

#### 範例 1: 回聲機器人

最簡單的機器人 - 回傳訪客所說的內容:

```
[Webhook 觸發器]
    ↓
[Function: 建立回應]
    return {
      response: "您說: " + $json.message.body
    };
    ↓
[HTTP 請求: 發送到 Odoo]
```

#### 範例 2: AI 支援機器人

使用 OpenAI 提供智慧回應:

```
[Webhook 觸發器]
    ↓
[OpenAI: 產生回應]
    提示: "您是一個有用的客戶支援人員。回答這個問題: {{ $json.message.body }}"
    ↓
[Function: 格式化回應]
    ↓
[HTTP 請求: 發送到 Odoo]
```

#### 範例 3: 智慧路由器

根據關鍵字路由到不同的工作流程:

```
[Webhook 觸發器]
    ↓
[Switch: 偵測意圖]
    情況 1: 包含「價格」 → [價格工作流程]
    情況 2: 包含「支援」 → [建立工單]
    情況 3: 包含「銷售」 → [通知銷售團隊]
    預設: [一般 FAQ 回應]
    ↓
[HTTP 請求: 發送到 Odoo]
```

### Webhook 中的可用資料

當 Odoo 向 n8n 發送 webhook 時,您會收到這些資料:

```javascript
{
  event_type: "message_received",
  timestamp: "2024-01-15T10:30:00Z",
  session: {
    id: 123,                          // 會話資料庫 ID
    uuid: "abc-123-def",              // 唯一會話識別碼
    name: "訪客 #45",                  // 會話顯示名稱
    started_at: "2024-01-15T10:25:00Z",
    visitor_name: "王大明",            // 訪客姓名(如果提供)
    visitor_country: "TW",            // 國家代碼
    visitor_lang: "zh_TW"             // 語言代碼
  },
  message: {
    id: 456,                          // 訊息資料庫 ID
    body: "您好,我需要協助",           // 訊息文字
    author_id: 789,                   // 作者合作夥伴 ID
    author_name: "王大明",            // 作者顯示名稱
    author_type: "visitor",           // "visitor" 或 "operator"
    created_at: "2024-01-15T10:30:00Z"
  },
  channel: {
    id: 1,                            // 頻道資料庫 ID
    name: "網站支援"                   // 頻道名稱
  },
  metadata: {
    odoo_base_url: "https://your-odoo.com",
    callback_url: "https://your-odoo.com/im_livechat_n8n/webhook",
    api_key_header: "X-Odoo-Livechat-API-Key"
  }
}
```

**使用範例**:

- `{{ $json.message.body }}` - 存取訊息文字
- `{{ $json.session.uuid }}` - 取得回應的會話 UUID
- `{{ $json.visitor_name }}` - 使用訪客姓名個性化回應
- `{{ $json.metadata.callback_url }}` - 動態回呼 URL

## 監控與日誌

### 檢視 Webhook 日誌

#### 從頻道設定

1. 前往**網站** → **即時聊天** → **頻道**
2. 選擇您的頻道
3. 點擊 **檢視 Webhook 日誌**按鈕
4. 日誌已篩選為僅顯示此頻道的活動

#### 從全域日誌檢視

1. 前往**網站** → **設定** → **Webhook 日誌**
2. 檢視所有頻道的所有 webhook 活動
3. 使用篩選器縮小範圍:
   - 方向(出站/入站)
   - 狀態(成功/失敗/逾時)
   - 時間段(今天、最近 7 天等)

### 了解日誌項目

每個日誌項目包含:

- **時間戳記**: webhook 觸發的時間
- **方向**:
  - **出站**: Odoo → n8n(訪客訊息發送到工作流程)
  - **入站**: n8n → Odoo(收到自動回應)
- **狀態**:
  - **成功**: Webhook 成功完成
  - **失敗**: Webhook 失敗(網路錯誤、無效回應等)
  - **逾時**: Webhook 花費超過 10 秒
- **回應時間**: 請求花費的時間(以毫秒為單位)
- **HTTP 狀態**: 回傳的狀態碼(200、404、500 等)
- **請求負載**: 發送的資料(點擊展開並檢視 JSON)
- **回應負載**: 收到的回應
- **錯誤訊息**: webhook 失敗時的詳細資訊

### 解讀常見的日誌模式

#### 健康系統
- 大多數日誌顯示「成功」狀態
- 回應時間低於 2000 毫秒
- HTTP 狀態 200

#### 網路問題
- 許多「逾時」狀態
- 高回應時間(>8000 毫秒)
- 檢查與 n8n 的網路連線

#### 設定問題
- 「失敗」狀態,401 錯誤
- 錯誤訊息:「無效的 API 金鑰」
- 解決方案: 驗證兩個系統中的 API 金鑰是否相符

#### n8n 工作流程問題
- 「失敗」狀態,500 錯誤
- 檢查 n8n 工作流程執行日誌的錯誤
- 常見問題: 缺少節點、無效表達式

### 日誌保留

- 日誌預設保留 **30 天**
- 自動清理透過排程動作每天執行
- 舊日誌會被永久刪除

**修改保留期限**:
1. 前往**設定** → **技術** → **自動化** → **排程動作**
2. 尋找「N8N Webhook 日誌清理」
3. 如需要,修改 Python 程式碼(變更 `timedelta(days=30)`)

## 疑難排解

### 問題: Webhook 未觸發

**症狀**: 在聊天中發送訊息,但 n8n 工作流程未觸發

**解決方案**:

1. **檢查是否已啟用整合**
   - 前往頻道設定 → N8N 整合分頁
   - 驗證「啟用 N8N 整合」是否已勾選

2. **驗證 Webhook URL**
   - 檢查 URL 是否正確且可存取
   - 使用 curl 測試:
     ```bash
     curl -X POST https://your-n8n.com/webhook/odoo-livechat \
       -H "Content-Type: application/json" \
       -d '{"test": true}'
     ```

3. **檢查防火牆/網路**
   - 確保 Odoo 伺服器可以連接到 n8n
   - 檢查防火牆規則
   - 驗證 DNS 解析

4. **檢視 Odoo 日誌**
   - 啟用除錯日誌: `--log-level=debug`
   - 搜尋: `odoo.addons.im_livechat_n8n`

### 問題: n8n 未收到 Webhook

**症狀**: Odoo webhook 日誌顯示「成功」,但 n8n 沒有執行

**解決方案**:

1. **檢查 Webhook 路徑**
   - 驗證 n8n 中的路徑是否與 Odoo 中的 URL 相符
   - 路徑區分大小寫

2. **檢查 Webhook 是否啟動**
   - n8n 工作流程必須**啟動**
   - 檢查工作流程狀態切換

3. **檢視 n8n 日誌**
   - 檢查 n8n 執行日誌的錯誤
   - 尋找 webhook 驗證問題

### 問題: 回應未出現在聊天中

**症狀**: n8n 工作流程執行,但訊息未出現在 Odoo 聊天中

**解決方案**:

1. **驗證 API 金鑰**
   - 檢查 n8n HTTP 請求節點中的 API 金鑰是否與 Odoo 相符
   - 金鑰區分大小寫
   - 無額外空格

2. **檢查會話 UUID**
   - 確保您使用正確的 UUID: `{{ $json.session.uuid }}`
   - UUID 必須來自傳入的 webhook

3. **驗證請求格式**
   - 檢查 Content-Type 標頭: `application/json`
   - 檢查 X-API-Key 標頭是否存在
   - 驗證 JSON 結構是否符合文件

4. **檢查 Webhook 日誌**
   - 前往 Odoo webhook 日誌
   - 依「入站」篩選
   - 檢查錯誤訊息

5. **檢查會話是否啟動**
   - 會話可能已關閉
   - 訪客可能已離開聊天

### 問題: 高回應時間

**症狀**: Webhook 日誌顯示成功但回應時間 > 5000 毫秒

**解決方案**:

1. **最佳化 n8n 工作流程**
   - 減少外部 API 呼叫
   - 儘可能使用快取
   - 簡化複雜邏輯

2. **檢查 n8n 伺服器資源**
   - CPU/記憶體使用率
   - 網路延遲
   - 資料庫效能

3. **使用非同步處理**
   - 對於複雜的工作流程,發送立即確認
   - 在背景處理
   - 準備好時發送後續訊息

### 問題: 訊息被截斷

**症狀**: 長訊息被截斷或拒絕

**解決方案**:

1. **檢查訊息大小**
   - 最大大小: 10KB(10,240 位元組)
   - 將長訊息分割成多個部分

2. **驗證編碼**
   - 確保 UTF-8 編碼
   - 注意特殊字元

### 錯誤參考

| 錯誤代碼 | 意義 | 解決方案 |
|---------|------|---------|
| 400 | 無效的負載 | 檢查 JSON 結構、必填欄位 |
| 401 | 無效的 API 金鑰 | 驗證 Odoo 和 n8n 中的 API 金鑰是否相符 |
| 404 | 找不到會話 | 會話可能已關閉或 UUID 不正確 |
| 500 | 伺服器錯誤 | 檢查 Odoo 日誌的堆疊追蹤 |
| 逾時 | 請求花費 > 10 秒 | 最佳化 n8n 工作流程或增加逾時 |

## 常見問題

### 一般問題

**問: 我需要付費的 n8n 帳戶嗎?**
答: 不需要,n8n 有免費的自行託管選項。也有雲端方案可供選擇。

**問: 我可以在多個即時聊天頻道中使用嗎?**
答: 可以!每個頻道都可以有自己的 webhook URL 和 API 金鑰。

**問: 客服人員的訊息會發送 webhook 嗎?**
答: 不會,只有訪客訊息會觸發 webhook 以避免迴圈。

**問: 如果 n8n 當機會發生什麼?**
答: Odoo 將以指數退避重試 3 次。聊天繼續正常運作。

**問: 我可以停用日誌記錄嗎?**
答: 日誌記錄是內建的,用於除錯。您可以設定較短的保留期限。

### 技術問題

**問: 這個整合是即時的嗎?**
答: 是的,當訪客發送訊息時,webhook 會立即觸發。

**問: Webhook 是否會阻塞?**
答: 不會,webhook 在背景執行緒中發送,以避免延遲聊天。

**問: 我可以在回應中發送豐富的格式嗎?**
答: 可以,在訊息主體中使用 HTML(例如 `<b>粗體</b>`、`<i>斜體</i>`)。

**問: 失敗時會重試幾次?**
答: 3 次自動重試,指數退避(1 秒、2 秒、4 秒延遲)。

**問: Webhook 逾時是多少?**
答: 每次嘗試 10 秒,重試總計 30 秒。

**問: 我可以為每個觸發發送多個回應嗎?**
答: 可以,在多個 HTTP 請求中使用相同的會話 UUID。

### 安全問題

**問: API 金鑰安全嗎?**
答: 是的,它使用密碼學安全的隨機令牌產生(32 位元組)。

**問: 我應該使用 HTTPS 嗎?**
答: 絕對要!在正式環境中始終使用 HTTPS。

**問: 我可以依 IP 限制 webhook 存取嗎?**
答: 可以,設定您的防火牆僅允許您的 n8n 伺服器 IP。

**問: 應該多久輪換一次 API 金鑰?**
答: 建議每 90 天或遭到洩露時輪換。

**問: 訊息是否已加密?**
答: 使用 HTTPS 時,所有流量都經過 TLS 加密。

### 工作流程問題

**問: 我可以觸發多個 n8n 工作流程嗎?**
答: 目前每個頻道一個 webhook URL,但您可以在 n8n 中路由到多個工作流程。

**問: 我可以使用條件邏輯嗎?**
答: 可以!使用 n8n 的 Switch 或 IF 節點根據訊息內容路由。

**問: 我可以與 AI 服務整合嗎?**
答: 可以!n8n 有 OpenAI、Anthropic、Google AI 等節點。

**問: 我可以從聊天建立 CRM 潛在客戶嗎?**
答: 可以!使用 n8n 的 HTTP 請求節點呼叫 Odoo 的 API 來建立潛在客戶。

**問: 我可以發送通知到 Slack/Discord 嗎?**
答: 可以!n8n 對大多數通知服務都有原生節點。
